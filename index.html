GitHub Pages使用入门

cosyer 2022-11-05 原文

1.什么是GitHub Pages

GitHub Pages可以当做项目的Websites，GitHub Pages有两种最基本的用法：

  • 作为你自己（或者组织）的网站（访问地址示例：http://username.github.io）
  • 作为你某一个项目的网站（访问地址示例：http://username.github.io/projectname）

2.创建方法

  • 创建一个仓库

以username.github.io 作为仓库名，其中username是github 的用户名，这个名字不能出错。如果出错，那么需要你删掉这个仓库重来，更改仓库名是无效的。

  • 为此仓库添加内容

直接在github网站上的此项目的仓库下直接点击 create new file（比如index.html），然后输入内容并保存。也可以将此项目从github克隆到本地，然后在本地添加要展示的内容，并 push 到远程仓库。

  • 访问

完成以上步骤，就可以在浏览器用 http://username.github.io 格式的网址访问网页了。 也可以在此仓库下新建项目http://username.github.io/projectname访问。
注意：在访问之前你可能需要等待一段时间，等待刷新 github 分配的新域名。 在此项目的 Setting 选项下边更改你的个性化设置，比如绑定你的个人域名、开启 HTTPS 等等。

3.创建新项目添加到Github Pages

  • 创建新仓库

  • 提交文件

这里需要注意的是选择提交到哪个分支，可以提交到 master分支，但也可以创建一个gh-pages分支，这样更新项目的网站就不会影响项目的开发。

  • 设置GitHub Pages的source

点击项目的Settings按钮，找到GitHub Pages 选项，然后选择相应的分支作为source，如果在上一步选择了创建gh-pages分支，那么这里就无需设置，因为 github 默认gh-pages 分支作为项目的GitHub Pages source，当你创建gp-pages分支的时候就会自动设置好。
